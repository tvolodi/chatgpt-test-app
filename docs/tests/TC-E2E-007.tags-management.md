# TC-E2E-007: Tag Management UI

## Description
Verify the end-to-end functionality of the Tag Management UI, including listing, creating, editing, and deleting tags.

## Pre-conditions
- User is authenticated (or authentication is mocked).
- Backend API is reachable (or mocked).

## Test Scenarios

### 1. View Tag List
- **Objective**: Verify that the tag list loads and displays data correctly.
- **Steps**:
  1. Navigate to `/dashboard/tags`.
  2. Verify the "Tags" heading is visible.
  3. Verify the "Create Tag" button is visible.
  4. Verify the table displays tags (mocked data: `tag1`, `tag2`).
- **Expected Result**: Tags are listed with correct names and codes.

### 2. Create New Tag
- **Objective**: Verify creating a new tag.
- **Steps**:
  1. Click "Create Tag".
  2. Fill in "Code" with `new-tag`.
  3. Fill in "Name (EN)" with `New Tag`.
  4. Click "Save".
- **Expected Result**:
  - Redirects back to the list.
  - (Optional) Success message is shown.
  - The new tag appears in the list (if mock is updated).

### 3. Edit Tag
- **Objective**: Verify editing an existing tag.
- **Steps**:
  1. Navigate to `/dashboard/tags/tag1`.
  2. Verify the form is pre-filled with `tag1` data.
  3. Change "Name (EN)" to `Updated Tag`.
  4. Click "Save".
- **Expected Result**:
  - Redirects back to the list.
  - The updated name is reflected.

### 4. Delete Tag
- **Objective**: Verify deleting a tag.
- **Steps**:
  1. In the list view, click "Delete" for a tag.
  2. Confirm the dialog.
- **Expected Result**:
  - The tag is removed from the list.

## Dependencies
- Playwright
- `web/src/app/components/tags`
